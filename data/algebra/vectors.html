Produit Scalaire (Dot Product)

Fondamental en IA !
$u \cdot v = ||u|| ||v|| \cos(\theta)$
Si le produit est grand, les vecteurs pointent dans la même direction (ils sont "d'accord").

Notation : Un vecteur colonne $\begin{pmatrix} x \\ y \end{pmatrix}$ est standard en algèbre.

📏 Norme d'un vecteur

Norme L2 (euclidienne)

\(\|\vec{v}\| = \sqrt{v_1^2 + v_2^2 + \cdots + v_n^2}\)

C'est la longueur du vecteur. On prend chaque composante, on l'élève au carré, on additionne le tout, puis on prend la racine carrée (\(\sqrt{}\)) de cette somme.

Exemple : Si \(\vec{v} = \begin{pmatrix} 3 \\ 4 \end{pmatrix}\), alors \(\|\vec{v}\| = \sqrt{3^2 + 4^2} = \sqrt{9 + 16} = \sqrt{25} = 5\).

🤖 ML : La norme L2 est utilisée comme régularisation Ridge pour empêcher les poids du modèle de devenir trop grands. On ajoute \(\lambda \sum w_i^2\) à la loss : les gros poids sont pénalisés quadratiquement.

Norme L1 (Manhattan)

\(\|\vec{v}\|_1 = |v_1| + |v_2| + \cdots + |v_n|\)

On additionne les valeurs absolues de chaque composante. Pas de carré, pas de racine. C'est la distance que parcourrait un taxi sur une grille (d'où le nom "Manhattan").

🤖 ML : La norme L1 (Lasso) favorise des poids exactement à zéro \(\rightarrow\) sélection automatique de features. Si un poids tombe à 0, la feature correspondante est ignorée par le modèle.

⚖️ L1 vs L2 en résumé :

  • L2 (Ridge) : poids petits mais rarement nuls. Bon quand toutes les features comptent.
  • L1 (Lasso) : poids souvent exactement 0. Bon pour la sélection de features.
  • Elastic Net : combine les deux (\(\alpha L1 + (1-\alpha) L2\)).

🧩 Combinaison linéaire

\(\vec{w} = \alpha \vec{u} + \beta \vec{v}\)

Tout nouveau vecteur créé en multipliant des vecteurs existants par des scalaires puis en les additionnant est une combinaison linéaire.

Exemple : Si \(\vec{u} = \begin{pmatrix} 1 \\ 0 \end{pmatrix}\) et \(\vec{v} = \begin{pmatrix} 0 \\ 1 \end{pmatrix}\), alors \(3\vec{u} + 2\vec{v} = \begin{pmatrix} 3 \\ 2 \end{pmatrix}\). On a "dosé" chaque direction.

🤖 ML : Un neurone calcule exactement une combinaison linéaire de ses entrées :

\(z = w_1 x_1 + w_2 x_2 + \cdots + w_n x_n + b\)

Les \(w_i\) sont les poids (scalaires), les \(x_i\) sont les features (vecteurs d'entrée), et \(b\) est le biais. Ensuite on applique une activation non-linéaire (\(\sigma\), ReLU, ...).

🌐 Espace engendré (Span)

\(\text{Span}(\vec{u}, \vec{v}) = \{ \alpha \vec{u} + \beta \vec{v} \mid \alpha, \beta \in \mathbb{R} \}\)

L'espace engendré (Span) est l'ensemble de tous les vecteurs qu'on peut créer par combinaison linéaire d'une famille de vecteurs.

Si \(\{\vec{u}, \vec{v}\}\) sont deux vecteurs non-colinéaires dans \(\mathbb{R}^2\), alors \(\text{Span}(\vec{u}, \vec{v}) = \mathbb{R}^2\) : on peut atteindre n'importe quel point du plan.

Mais si \(\vec{v} = 3\vec{u}\) (colinéaires), le span ne couvre qu'une droite. On a "perdu" une dimension.

🤖 ML : Les features d'un dataset définissent un sous-espace. Si deux features sont corrélées (quasi-colinéaires), elles n'ajoutent pas de dimension utile \(\rightarrow\) c'est pour ça que PCA réduit la dimensionnalité : elle identifie les directions qui "comptent vraiment" et élimine la redondance.

🔗 Indépendance linéaire

\(\alpha_1 \vec{v}_1 + \alpha_2 \vec{v}_2 + \cdots + \alpha_n \vec{v}_n = \vec{0} \implies \alpha_1 = \alpha_2 = \cdots = \alpha_n = 0\)

Des vecteurs sont linéairement indépendants si aucun d'entre eux ne peut être écrit comme combinaison linéaire des autres. La seule façon d'obtenir \(\vec{0}\) en les combinant, c'est de mettre tous les coefficients à zéro.

Intuition : Chaque vecteur apporte une information nouvelle, une direction qu'aucun autre ne couvre. Si on en retire un, on perd une dimension du span.

🤖 ML : Features indépendantes = maximum d'information. Features dépendantes = redondance \(\rightarrow\) multicolinéarité. Cela rend la régression linéaire instable (la matrice \(X^TX\) devient presque singulière) et les coefficients deviennent peu fiables.

💡 Astuce pratique : Pour détecter la multicolinéarité, calculez le VIF (Variance Inflation Factor) de chaque feature. Un VIF > 10 signale une forte dépendance. Solutions : retirer la feature redondante, ou utiliser PCA / régularisation.

🎯 Produit scalaire decortique

Le produit scalaire a deux formules equivalentes :

Formule algebrique

\(\vec{u} \cdot \vec{v} = u_1 v_1 + u_2 v_2 + \cdots + u_n v_n\)

Formule geometrique

\(\vec{u} \cdot \vec{v} = \|\vec{u}\| \, \|\vec{v}\| \, \cos\theta\)

Decortiquons chaque terme de la formule geometrique :

  • \(\|\vec{u}\|\) = longueur (norme) du vecteur \(\vec{u}\)
  • \(\|\vec{v}\|\) = longueur (norme) du vecteur \(\vec{v}\)
  • \(\cos\theta\) = cosinus de l'angle \(\theta\) entre les deux vecteurs

Que nous dit \(\cos\theta\) ?

→→

\(\cos\theta = 1\)

Meme direction

→↑

\(\cos\theta = 0\)

Perpendiculaires

→←

\(\cos\theta = -1\)

Opposes

🤖 ML : La similarite cosinus en NLP compare deux embeddings de mots :

\(\text{cos\_sim}(\vec{u}, \vec{v}) = \frac{\vec{u} \cdot \vec{v}}{\|\vec{u}\| \, \|\vec{v}\|}\)

"roi" et "reine" ont un cosinus proche de 1 (sens similaire). "roi" et "banane" ont un cosinus proche de 0 (aucun rapport). C'est la base de la recherche semantique et des systemes de recommandation.

🚀 Vecteurs en haute dimension

En 2D ou 3D, on peut dessiner un vecteur. Mais en ML, les vecteurs vivent dans des espaces a des centaines ou milliers de dimensions :

768

dimensions pour BERT

1536

dimensions pour les embeddings OpenAI

12288

dimensions pour GPT-4

💡 Intuition contre-intuitive : En haute dimension, presque tous les vecteurs aleatoires sont quasi-perpendiculaires entre eux ! Plus la dimension augmente, plus \(\cos\theta\) entre deux vecteurs aleatoires se concentre autour de 0. C'est un aspect de la malediction de la dimensionnalite.

🤖 ML : Consequences pratiques de la haute dimension :

  • Les distances euclidiennes deviennent peu discriminantes \(\rightarrow\) on prefere la similarite cosinus.
  • L'espace est si vaste qu'il faut enormement de donnees pour le "remplir" (curse of dimensionality).
  • Les techniques de reduction (PCA, t-SNE, UMAP) projettent les vecteurs en 2D/3D pour la visualisation.
